# Auto-logout on API 401

**Date:** 2026-07-06

## Context

Authenticated API calls go through `fetchWithAuth` in `lib/api/client.ts`. On 401 it retries once after `POST /auth/refresh`. If refresh fails, tokens are cleared but the user stayed on the current screen with error UI.

SWR fetchers (`getGuilds`, `getGroceryLists`) and imperative calls (`createGrocery`, `deleteGroceriesBatch`, `logoutSession`) all use this client.

## Problem

When refresh fails (revoked session, expired refresh token, etc.), the app had no session but still showed authenticated routes. The user had to discover the logout button manually.

## Decision

Register an auth-expired callback from the React layer and invoke it from `fetchWithAuth` when:

1. No valid access token can be obtained (including failed refresh), or
2. A 401 response remains after a failed refresh retry.

`app/_layout.tsx` subscribes via `onAuthExpired`, clears tokens, and `router.replace('/(auth)/login')`.

## Why `fetchWithAuth` stays outside React

It centralizes auth headers, token refresh, and retry logic for all callers. SWR and similar libraries expect a plain async fetcher function, not a hook.

## Why a global callback instead of `useAppSWR`

A SWR wrapper would only cover hook-based reads. Imperative mutations (create/delete groceries) call the API client directly from event handlers and would still need separate 401 handling. The callback fires once at the shared transport layer for every authenticated request.

## Alternatives considered

1. **Check 401 in every SWR `onError` and catch block** — rejected; duplicates logic at every call site.
2. **`useAppSWR` plus a mutation wrapper** — rejected; two mechanisms instead of one, still easy to miss a path.
3. **Move fetch logic into a React hook** — rejected; imperative calls need the same refresh behavior without being inside a component render.

## Verification

When refresh fails on any authenticated request, tokens are cleared and the app navigates to `/(auth)/login`. Explicit sign-out and offline read-only behavior are unchanged.
